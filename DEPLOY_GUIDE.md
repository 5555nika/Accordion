# Инструкция: Как запускать и публиковать Vite + TypeScript проекты на GitHub Pages без нервов

Эта шпаргалка поможет вам настроить и опубликовать любой новый проект на Vite и TypeScript с нуля.

---

## 💡 Главный секрет: почему не работает просто так?
Обычные браузеры **не понимают** файлы TypeScript (`.ts`). 
Поэтому мы не можем просто загрузить исходные файлы на GitHub Pages.

Нам нужна **сборка**:
1. Команда сборки (`npm run build`) компилирует TypeScript в чистый JavaScript и оптимизирует CSS.
2. Все готовые к запуску в браузере файлы складываются в папку **`dist`**.
3. Именно содержимое папки **`dist`** мы и должны публиковать на GitHub Pages.

---

## 🛠️ Шаг 1. Создание нового проекта
Создайте проект стандартным способом в терминале:
```bash
# 1. Создаем проект (выберите Vanilla и TypeScript)
npm create vite@latest my-new-project

# 2. Переходим в папку проекта
cd my-new-project

# 3. Устанавливаем все зависимости
npm install
```

---

## 📦 Шаг 2. Подготовка проекта к публикации (Делается 1 раз для проекта)

Чтобы сайт корректно работал и публиковался одной командой, нужно настроить три вещи:

### 1. Установите утилиту для деплоя
В терминале внутри папки проекта введите:
```bash
npm install gh-pages --save-dev
```

### 2. Добавьте скрипт деплоя в `package.json`
Откройте файл `package.json` в редакторе кода и найдите секцию `"scripts"`. Добавьте туда команду `"deploy"`:
```json
"scripts": {
  "dev": "vite",
  "build": "tsc && vite build",
  "preview": "vite preview",
  "deploy": "npm run build && gh-pages -d dist"  // <-- ДОБАВИТЬ ЭТУ СТРОКУ
}
```

### 3. Укажите относительные пути в `vite.config.ts`
Откройте файл `vite.config.ts` в корне проекта. Он должен выглядеть так (добавьте строчку `base: './'`):
```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  base: './', // <-- ОЧЕНЬ ВАЖНО: делает все пути относительными
})
```

---

## 🚀 Шаг 3. Связь с GitHub и первая отправка (Делается 1 раз для проекта)

1. Перейдите на GitHub и создайте **абсолютно новый пустой репозиторий** (не ставьте галочки "Add README" или ".gitignore").
2. Скопируйте ссылку на репозиторий (например, `https://github.com/ваш_логин/имя_репозитория.git`).
3. В терминале вашего проекта выполните по очереди следующие команды:

```bash
# Инициализируем локальный Git
git init

# Добавляем все файлы проекта
git add .

# Делаем первый коммит
git commit -m "feat: инициализация проекта"

# Создаем ветку main
git branch -M main

# Связываем локальную папку с вашим репозиторием на GitHub
git remote add origin ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ

# Отправляем исходный код на GitHub
git push -u origin main
```

---

## 🌐 Шаг 4. Публикация сайта на GitHub Pages

Теперь, когда всё настроено, публикация выполняется **одной командой**:

1. В терминале проекта введите:
   ```bash
   npm run deploy
   ```
   *Эта команда автоматически соберет проект, создаст папку `dist` и отправит ее на GitHub в ветку `gh-pages`.*

2. Зайдите в ваш репозиторий на сайте GitHub:
   * Перейдите в **Settings** (Настройки) ➔ **Pages** (в левом меню).
   * В разделе **Build and deployment** найдите **Source** и выберите **Deploy from a branch**.
   * В поле **Branch** выберите ветку **`gh-pages`** и нажмите **Save**.

---

## 🔄 Шаг 5. Как обновлять проект в будущем?

Когда вы измените код на компьютере и захотите обновить сайт:
1. Отправьте изменения исходного кода в основную ветку:
   ```bash
   git add .
   git commit -m "Что изменилось"
   git push
   ```
2. Опубликуйте новую версию сайта одной командой:
   ```bash
   npm run deploy
   ```
   *Сайт на GitHub Pages обновится автоматически в течение минуты!*
